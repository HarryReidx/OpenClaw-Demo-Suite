# Demo Flow

这份文档用于公司内部宣讲时的讲解顺序。建议直接从 `http://你的局域网IP:8000/` 进入总入口，然后按编号依次演示。

## 01. 最简单的 LLM 一问一答

建议话术：

- 先让大家看到，大模型最基础的形态就是“输入一段自然语言，返回一段自然语言”。
- 这一层还不是 Agent，只是一次性的推理与生成。
- 适合解释 Prompt、系统提示词和模型能力边界。

## 02. 带记忆的多轮对话

建议话术：

- 让大家观察模型为什么能“记得上一句”。
- 解释所谓记忆，本质上是把历史消息一起带给模型，或者存储后按需召回。
- 这一层开始更像聊天产品，但还没真正拥有外部行动能力。

## 03. 可写文件的 Agent

建议话术：

- 这里开始进入 Agent 概念：不只是回答，而是“调用工具做事”。
- 可以解释 Tool Calling、Function Calling 和本地执行器的关系。
- 演示“帮我写一份发布通知”、“生成一段活动文案”，让 Agent 把结果落到文件里。

## 04. 联网检索并生成网页报告

建议话术：

- 说明 Agent 为什么要联网：很多问题只靠模型参数记忆是不够的。
- 展示检索、摘要、结构化输出、网页交付这条完整链路。
- 这一站最适合讲 Search、RAG、Report Generation 的区别和协作方式。

## 05. 移动端 OpenClaw 风格对话

建议话术：

- 这一层开始像真实产品，而不只是技术演示。
- 打开手机页面，让大家看到同一个 Agent 在移动端也能工作。
- 上传一张图片，现场演示多模态理解，让大家直观看到“看图 + 对话”的统一体验。

## 06. AI 资讯定时推送

建议话术：

- 这是 OpenClaw 很重要的一类能力：不等人来问，而是让任务自己跑。
- 强调自动化、定时任务、摘要分发和企业微信推送这些能力如何组合。
- 可以告诉大家：这就是从“会聊天的模型”变成“持续工作的 Agent”的关键一步。

## 07. AI 知识库 RAG

建议话术：

- 这一站最适合解释为什么企业场景不能只靠模型参数记忆。
- 展示先上传文档、再检索相关片段、最后基于片段回答的完整过程。
- 可以和 04 做对比：一个面向外部互联网，一个面向内部知识库。
